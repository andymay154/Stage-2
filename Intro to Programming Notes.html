Basics

The internet is full of lots of servers hosting web pages which are basically HTML (HyperText Markup Language) documents my browser can interpet and present to me.
HTML is made up of Tags, Attributes and Elements. Some tags are inline and some tags are block, the latter of which essentially puts a 'box' around the content allowing for additional specific formatting. DIV is an example of a block element, whereas SPAN is an example of an inline element, but both are container elements. Usually an opening TAGs will need a closing TAG except in the case of void tags such as "img" or "br", void tags do not require a corresponding closing tag.

HTML Tags can reference text, images, audio, video or even just plain old text - all filetypes can exist on the internet!

Boxes
The Box Model

When creating a structured document you must first divide everything up into boxes and define those boxes with DIV tags. You can also give each element a CLASS attribute so that you can easily apply styles afterwards. Tag's can belong to multiple Classes.

Boxes have 4 elements:

  1. Margin
  2. Border
  3. Padding
  4. Content

The code box-sizing:border-box will allow you to set the size of the total box including the border and margin, which makes it much easier to determine the actual size of the box rather than just the content. In addition, you can choose to set the size in terms of percentage of screen rather than defined pixel count!

Box Positioning

DIV's are BLOCK elements, so by default they will take up the whole width of the page. Adding the rule "display:flexi;" to the appropriate CSS will override this behaviour and let DIV's appear next to each other.

When creating web sites the procedue to follow is: Code --> Test --> Refine.

  1. Look for narural boxes
  2. Look for repeated styles
  3. Write your HTML
  4. Apply Styles (biggest to smallest)
  5. Fix things

Variables, Strings and Functions
Variables

Using a Veriable you can create a name and assign a value to that name. The value of the variable can change depending on what you assign to it, and that assignement may change during code execution i.e. "variables can vary". You use the name of the variable within your code statements, and the execution result will look at the value currently held by that variable.
You assign a variable by: Name = Expression.

Also remember that when working with programming languages, "=" does not mean equal to. Instead it means "asssignment"! In addition, common practice suggests that variables should never start with a Capital letter.

Strings

A string is a sequence of characters and can be enclosed between either single or double quotes, the only stipulation is that you must always close the string with the same option you take to open the string with. Without quotes Python will see the text as a variable instead of a string which will cause an error! Interestingly it is actually possile to multiply stings, which will actually just print the string the number of times you specify i.e. 12 * "!" will result in "!!!!!!!!!!!!".
To comment code in Python, prefix the line with a "#" character.

Something really important you can do with strings is index them and work with them based on that indexing to extract just parts of the string. Each character is essentially given a number, beginning at 0 for the first.

string[expression] = One character string e.g. 'udacity'[0] = 'u' string[expression:expression] = from start to stop positions (stop will be -1) e.g. 'udacity'[3:5] = 'ci'

.find is a method that can be used on strings; a method ia really just a procedure. This procedure will happen to find the startinn point of the string you are trying to find within the target string. Note that it will only ever find the first occurence and an output of -1 will mean that the string was not found. In addition, remember that the .find methos is case sensitive.

You can also pass in a parameter to the .find method in order to specify the starting position, and you can store the position result in a variable. This will allow you to find additional ocurances of the string that you are searching for.

For example:

first = example.find('!')
second = example.find('!',first+1)

Using triple quotation marks will allow you to specify a string over multiple lines. For example:

a sting

multiple lines!

Input, Function, Output

When defining a function you will specify the name of the procedure and then within parentheses you will specifiy the name(s) of the inpute to the function. Next you will specity the return value of the procedure, for example:

def rest_of_string(s): return s[1:]
This procedure will simply take a string as an input and output the same string with the first letter removed.
Remember that any variable assignments within a function will apply within that function only, to use it outside of the funtion the result value must be included in a return value or an output. If there is no return value or output, then the function will actually do nothing at all...well, nothing that you can see anyway!

A function is designe to take one or more inputs, performs a desired action upon those inputs and then produce an Output. For example, the action could be to concatenate 3 strings and you would pass in those strings to the function. For example:

def concat(string1, string2, string3): return string1+string2+string3
When using the procedure you would provide the strings to concatenate, such as:
concat(Add, These, Together); this would output "AddTheseTogether"

The purpose of functions is simply to save time coding repetitive actions or repeating code, you can call the function as and when you need it throughout your application.

When using example code always test it to make sure it works properly to begin with. If you have problems and need to debug your own code, just comment it out to begin with so when you get it working you will be able to see where you went wrong to begin with and learn from it.

  1. Examine error messages when programs crash

Lists

A list can store a number of items separated by a comma.

< list> [< expression>,< expression>,....]

Remember that lists are orders, so it is important you place items in the order you want them.
Lists can contain more lists and the data types can be mixed, where as strings can only contain characters.
Remember that the index of the list always starts from 0.

Lists provide us with a way of putting data into an ordered structure and are mutable whereas strings are not. What they basically means is that you can change elements within a list but not a string. However, be careful because if more than one variable references the same list, changing (mutating) the list using one variable will also change the list for all other variables that reference the same list!

To change a value in a list:
p = 'H', 'e', 'l', 'l', 'o'
p[0] = Y
now p = 'Y', 'e', 'l', 'l', 'o'

Two different ways of referencing the same object is called 'Aliasing'.

When altering lists within a procedure, you do no need to 'return' the result, since you will be directly mutating the list itself. The reason this does not work with string assignments is that strings are non-mutable i.e. you can't change it, so instead you change the value that the new variable in the procedure refers to, which in turn means you need to return that value from the procedure.

Append will create a new element on the end of an existing list and reference the appended object. That object could be a single element, but it could equally be another list. Either way, it will only result in ONE additional element on the end of the original list.

To append:
< list>.append(< element>)

Append is a method that will mutate the list it is invoked upon. It is not creating a new list and you do not need to 'return' anything, the list itself will be modified.

Using "+" will actually concatenate the objects and create a new object, so the concatenation of two lists with two elements each, will result in a new list containing 4 elements. If you use this in a procedure then you would need to return the result of the concatenation otherwise nothing will be changed outside of the procedure itself.

len will take a list as its input and output the number of elements in that list; it will only count the outer elements.

While and For Loops

A while loop will continue to cycle as long as the condition is True. For example:

----While [Test Expresstion]:
--------Do Something until test is false
----Return

Can also use a "For Loops". These are similar to While loops, but instead of looking for a text condition it will repeat for every element found in the list.
< for> element in < list>:
----< block>

example:

def sum_list(numbers):
i = 0
for item in numbers:
i = i + item
return i

print sum_list([1, 7, 4])
#>>> 12

Elements in a list will have an index location, starting with 0.
< list>.index(< value>)
If the value is in the list it will return the index location of the first instance of that value, otherwise it will return an error.

< value> in < list>
Or could use "not in"
If the value is in the list the output is True, otherwise it is False.
